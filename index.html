Project 1: Malware Analysis Sandbox

Objective: This project focuses on setting up a secure and controlled environment for malware analysis using FlareVM and REMnux. The sandbox enables both static and dynamic analysis while ensuring network isolation to prevent unintended infections. The goal is to study how malware behaves, interacts with the system, and communicates over the network.

Environment Setup:

  • FlareVM (Windows VM): Pre-configured with reverse engineering and malware analysis tools.
  • REMnux (Linux VM): Used for network forensics and malware communication analysis.
  • VirtualBox: Provides an isolated virtualization environment.
  • INetSim: Simulates various internet services to observe malware network activity.

Methodology:

  1. Setting Up Virtual Machines:
    • Installed and configured FlareVM and REMnux in VirtualBox.
    • Assigned a host-only network adapter to ensure network isolation.
  2. Static Analysis:
    • Scanned the malware sample using VirusTotal to check for known threats.
    • Analyzed the file structure using PEview, PEiD, and PE Studio.
    • Extracted hidden strings using FLOSS to identify potential commands and functions.
  3. Dynamic Analysis:
    • Executed the malware within FlareVM and monitored its behavior.
    • Used Procmon to track file system, registry, and process changes.
    • Checked system modifications using CFF Explorer.
    • Examined active network connections with TCPView.
  4. Network Traffic Monitoring:
    • Configured INetSim on REMnux to simulate internet services.
    • Captured network traffic using Wireshark to analyze HTTP, DNS, and other protocol interactions.

Tools & Technologies Used:

  • Malware Analysis Environments: FlareVM, REMnux
  • Virtualization: VirtualBox
  • Static Analysis Tools: VirusTotal, FLOSS, PEview, PEiD, PE Studio
  • Dynamic Analysis Tools: Procmon, CFF Explorer, TCPView
  • Network Analysis Tools: INetSim, Wireshark

Expected Outcomes:

  • Successfully executed and monitored malware in a secure, isolated environment.
  • Identified malware persistence techniques and behavioral patterns.
  • Gained insights into how malware interacts with system resources and networks.
  • Documented findings and mitigation strategies in a detailed report.
